Install the Angular CLI

You use the Angular CLI to create projects, generate application and library code, and perform a variety of ongoing development tasks such as testing, bundling, and deployment. To install the Angular CLI, open a terminal window and run the following command:
npm install -g @angular/cli






























Create a workspace and initial application

You develop apps in the context of an Angular workspace.

  • Run the CLI command ng new and provide the name my-app, as shown here:
    • ng new my-app
  • The ng new command prompts you for information about features to include in the initial app. Accept the defaults by pressing the Enter or Return key.

The Angular CLI installs the necessary Angular npm packages and other dependencies. This can take a few minutes.

The CLI creates a new workspace and a simple Welcome app, ready to run.

Run the application

The Angular CLI includes a server, for you to build and serve your app locally.

  • Navigate to the workspace folder, such as my-app
    • cd my-app
    ng serve --open
  • Run the following command:

The ng serve command launches the server, watches your files, and rebuilds the app as you make changes to those files.

The --open (or just -o) option automatically opens your browser to http://localhost:4200/.

Installing Flask and hosting the backend:

Before we start, we need to get a text file including all requirements for this project from the virtual environment and flask that is already run our own PCs. Later we use this file to install whatever that is required on the server automatically. To do so, we need to run the following command in our local virtual environment:
Pip freeze > requirements.txt

The requirements.txt file should be added to our project
Then we go to the host website (https://www.pythonanywhere.com/) and log in
In the “Dashboard” we choose “Files”
Then we add all the folders and files exactly how we had made them locally in the host
After that we return to the dashboard, and we choose “Web apps”
We choose the default name for the web app and select the appropriate python version to run it. (In my case it was Python 3.8)
After we create the web app, we go to the “Code” section and choose the “source code” which allows us to choose the path of the files we have uploaded before on the server.
Now we have to setup a virtual environment on the server. In order to do that we go back to the dashboard and choose “$ Bash” under the “Consoles” section. Then we enter the command below to create the virtual environment called my_virtualenv:
Mkvirtualenv –python=/usr/bin/python3.8 my-virtualenv
 After installation has been done, we install flask on the server using the following command:
pip install flask
 The we move into our project folder (in my case the name of the folder is cps530) and use the requirement.txt we explained before to install all requirement packages, libs, and other files.
cd cps530
 pip install -r requirements.txt
 Then we go back to the web app tab from dashboard and go to the “Virtualenv” section.
We give our virtual environment’s name (In my case it was my-virtualenv) and save it.
Next we go the “WSGI configuration file” under the “Code” section and we delete all the codes there except the flask part and then uncomment the flask part of the code. We change the value of the variable “path” to the path where our project exists in the server (in my case it is ‘/home/arash2079/cps530’). Also, we need to change “main_flask_app_file” in the following code to the name of the python app we wrote(In my case it was server.py) and “app” to the variable you put “Flask(__name__)” in it (mine is called app)
From main_flask_app_file import app as application
 The result for me will be:
From server.py import app as application
 Then we go back to the web app page and click on the “Reload arash2079.pythonanywhere.com” button
And now our backend is successfully hosted on the server.
